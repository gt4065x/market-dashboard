📈 미장 분석 & 국내 주식 추천 시스템

완전 자동화 구축 가이드 · GitHub Pages + n8n

무료 호스팅 코딩 불필요 매일 자동 업데이트

⏰ n8n 스케줄러 (매일 오전 8시)
   ↓
🤖 Claude API (미장 분석 + 종목 추천 생성)
   ↓
📄 data.json (GitHub Repository에 자동 업로드)
   ↓
🌐 GitHub Pages (index.html이 data.json을 읽어 화면 표시)

STEP 1 · GitHub 저장소 만들기

STEP 01

① GitHub.com 접속 후 새 저장소 생성

  1. github.com 로그인 → 우측 상단 + → New repository
  2. Repository name: market-dashboard
  3. ✅ Public 선택 (GitHub Pages 무료 사용)
  4. ✅ Add a README file 체크
  5. Create repository 클릭

② 파일 2개 업로드

저장소 페이지에서 Add file → Upload files

파일명내용
index.html대시보드 웹페이지
data.json초기 데이터 (n8n이 매일 덮어씀)
💡 위 가이드와 함께 제공된 index.html, data.json 파일을 그대로 업로드하세요.

STEP 2 · GitHub Pages 활성화

STEP 02
  1. 저장소 → Settings 탭
  2. 왼쪽 메뉴 → Pages
  3. Source: Deploy from a branch
  4. Branch: main / / (root) 선택
  5. Save 클릭
  6. 약 1~2분 후 URL 확인: https://YOUR_USERNAME.github.io/market-dashboard
⚠️ YOUR_USERNAME을 본인 GitHub 아이디로 변경하세요.

STEP 3 · GitHub Personal Access Token 발급

STEP 03

n8n이 data.json을 자동으로 업데이트하려면 GitHub API 인증이 필요합니다.

  1. GitHub → 우측 상단 프로필 → Settings
  2. 왼쪽 하단 Developer settings
  3. Personal access tokens → Tokens (classic)
  4. Generate new token (classic)
  5. Note: market-dashboard-bot
  6. Expiration: No expiration
  7. Scopes: ✅ repo 체크
  8. Generate token → 토큰 복사 (다시 볼 수 없음!)
⚠️ 토큰은 반드시 어딘가에 저장해두세요. 페이지를 벗어나면 다시 볼 수 없습니다.

STEP 4 · n8n 워크플로우 설정 (Zeabur)

STEP 04

① n8n 접속

Zeabur에서 실행 중인 n8n 인스턴스 접속 (기존 URL 사용)

② 워크플로우 import

  1. n8n 좌측 메뉴 → Workflows → Add Workflow
  2. 우측 상단 메뉴(⋮) → Import from file
  3. 제공된 n8n-workflow.json 파일 선택

③ 노드별 설정

🤖 Claude AI 분석 노드

Headers에 추가:
x-api-key: sk-ant-YOUR_ANTHROPIC_API_KEY

📤 GitHub PUT 노드

URL 수정:

https://api.github.com/repos/YOUR_USERNAME/market-dashboard/contents/data.json

Headers 수정:

Authorization: token YOUR_GITHUB_PAT

④ SHA 자동 조회 노드 추가 (중요!)

GitHub API로 파일을 업데이트할 때 현재 파일의 SHA가 필요합니다.
📤 GitHub PUT 노드 앞에 HTTP Request 노드를 추가하세요:

Method: GET
URL: https://api.github.com/repos/YOUR_USERNAME/market-dashboard/contents/data.json
Headers: Authorization: token YOUR_GITHUB_PAT

// 이후 PUT 노드의 sha 필드에 아래 expression 사용:
{{ $json.sha }}
💡 완성된 워크플로우 흐름: 스케줄러 → GET SHA → Claude 분석 → JSON 파싱 → GitHub PUT

STEP 5 · 워크플로우 테스트 & 활성화

STEP 05
  1. Test workflow 버튼으로 수동 실행
  2. 각 노드가 녹색(✅)이면 성공
  3. GitHub 저장소에서 data.json이 업데이트됐는지 확인
  4. GitHub Pages URL에서 대시보드 확인
  5. 우측 상단 토글로 워크플로우 Active 전환

실행 스케줄

cron의미
0 8 * * 1-5평일 오전 8시 (KST 기준으로 서버 시간 맞춤)
0 23 * * 0-4미국 장 마감 후 (UTC 기준 평일 밤 11시)
⚠️ Zeabur 서버가 UTC 기준이면 KST = UTC+9 를 고려해 cron 시간 조정 필요
